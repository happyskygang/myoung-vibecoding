1) 서비스 목표
	•	코드(특히 AI/ML) 문제를 제시하고
	•	유저가 풀이(코드 제출/실행/검증) 하며
	•	리더보드·토론·코드공유·학습콘텐츠로 실력 향상과 커뮤니티를 만드는 “코드 챌린지 플랫폼”

⸻

2) 핵심 요구사항(기능 목록)

A. 챌린지/문제(Competition / Challenge)
	•	문제 목록/검색/필터(난이도, 태그, 기간, 주최, 데이터 유형)
	•	문제 상세
	•	목적/평가지표(예: F1, RMSE), 데이터 설명, 제출 포맷, 규칙, 베이스라인
	•	일정(시작/종료), 리더보드 링크, 토론 링크, 코드 공유 링크
	•	참가(Join) / 팀 생성·초대(옵션)
	•	데이터 다운로드(권한/동의/라이선스 체크 포함)
	•	공지사항(룰 변경, 베이스라인 업데이트 등)

B. 제출/채점(Submission & Evaluation)
	•	제출 업로드(파일/깃 리포/컨테이너 이미지 등) + 제출 횟수 제한
	•	자동 채점 파이프라인
	•	제출물 검증(컬럼, row count, schema, 누락 값 등)
	•	스코어 계산
	•	결과 저장 및 리더보드 반영(공개/비공개 점수, public/private split 옵션)
	•	제출 히스토리(점수, 시간, 로그, 제출물 버전)
	•	부정행위/누수 방지(옵션)
	•	private test, 재채점, 코드 실행 제한, 데이터 접근 제어

C. 리더보드/랭킹/티어
	•	챌린지별 리더보드(정렬, 팀/개인, 최종/최고점)
	•	유저 프로필
	•	티어/XP, 참가 이력, 메달/배지, 공개 코드/노트북
	•	시즌 랭킹(옵션), 태그별 랭킹(옵션)

D. 커뮤니티(Discussion / Code Share)
	•	토론 게시판(질문, 공지, 팁, 버그리포트)
	•	코드 공유(노트북/레포 링크/첨부)
	•	좋아요, 댓글, 북마크
	•	신고/모더레이션(스팸, 저작권, 욕설)

E. 학습(Projects / Courses)
	•	“프로젝트 기반 튜토리얼”
	•	단계별 미션, 자동 채점(또는 퀴즈), 완주 배지
	•	랭커 특강/강의(영상/문서) + 추천 경로
	•	베이스라인 템플릿(예: tabular/NLP/CV) 제공

F. 운영/주최자(Host/Admin)
	•	챌린지 생성/수정(규칙, 데이터, metric, 기간)
	•	평가 설정(public/private, daily limit, ensemble 허용 등)
	•	리더보드 관리(실격 처리, 재채점)
	•	공지/FAQ 관리
	•	통계 대시보드(참여 수, 제출 수, 유입, 완주율)

G. 계정/권한/결제(옵션)
	•	로그인(Email/OAuth/SAML)
	•	역할: guest / user / host / admin / moderator
	•	유료 기능(프리미엄 강의, 기업 전용 대회, 더 많은 제출권 등) — 선택

4) 시스템 설계(아키텍처 개요)

서비스 구성(권장 분리)
	1.	Web/App(Frontend): Next.js/Vue 등
	2.	API Gateway / Backend
	•	Auth, Challenge, Submission, Leaderboard, Community, Learn
	3.	채점 워커(Evaluation Worker)
	•	큐 기반 비동기 처리(RabbitMQ/SQS/Kafka 등)
	•	컨테이너 격리 실행(Docker/K8s Job)
	4.	스토리지
	•	Object Storage(S3/MinIO): 데이터셋, 제출 파일, 아티팩트
	•	DB(PostgreSQL): 유저/챌린지/제출/랭킹/게시글
	•	Cache/Rank Store(Redis): 리더보드 캐시/세션/레이트리밋
	5.	관측/운영
	•	로그/메트릭/트레이싱(ELK, Prometheus, OpenTelemetry)

제출-채점 플로우
	•	(1) 유저 제출 업로드 → API가 파일을 Object Storage에 저장
	•	(2) API가 “채점 작업”을 큐에 발행(job_id)
	•	(3) 워커가 job을 가져와 격리 환경에서 실행
	•	제출물 검증 → metric 계산 → 결과/로그 생성
	•	(4) 결과를 DB에 저장하고 리더보드 업데이트
	•	(5) 프론트는 job 상태를 폴링/웹소켓으로 표시

보안/안정성 핵심
	•	컨테이너 실행 제한: CPU/메모리/시간/네트워크 차단(제출 코드 실행형일 경우)
	•	데이터 접근 제어: 대회 참가 동의 후 다운로드, 토큰 만료
	•	리더보드 일관성: “최고 점수만 반영” 정책, 재채점 기능
	•	레이트 리밋: 제출 횟수/분당 요청 제한

6) MVP 범위(2단계로 쪼개기)

MVP(가장 먼저)
	•	챌린지 리스트/상세
	•	데이터 다운로드
	•	제출 업로드(파일 기반) + 자동 채점(간단 metric)
	•	리더보드
	•	토론 게시판(기본 CRUD)
	•	로그인/프로필(기본)

V1 확장
	•	코드 공유, 좋아요/북마크
	•	티어/XP/배지
	•	팀 기능
	•	학습(프로젝트 미션형) + 수료 배지
	•	Host Console(대회 개설/관리)
	•	격리 실행/부정행위 대응 강화
